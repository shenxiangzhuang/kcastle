# Evals

## Design evals

Write 2-5 realistic prompts covering normal usage and at least one edge case.
For each prompt, note what the agent should produce and how to tell if it succeeded.

## Run and review

For each prompt:

1. Read the target skill's `SKILL.md` and follow its instructions to complete the task.
2. Compare the output against expected behavior.
3. Note what passed, what failed, and what needs the user's judgment.
4. Present a short summary to the user.

## Iterate

After each round of feedback:

1. Patch the skill to address specific failures.
2. Re-run the same prompts to check for regressions.
3. Stop when the user is satisfied or edits no longer improve quality.
